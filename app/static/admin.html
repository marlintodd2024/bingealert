⏱️ Smart Batching Settings

                

                    The portal automatically groups multiple episodes into single emails to prevent notification spam. 
                    When an episode downloads, it waits for more episodes and intelligently batches them together.
                

                
                

                    How it works:

                    
                        1. Episode downloads → Wait Initial Delay for Plex to index

                        2. Check Sonarr queue → More episodes coming? Extend by Extension minutes

                        3. Keep checking every Check Frequency seconds

                        4. Send batch when queue empty OR hit Max Wait time

🔍 Quality & Release Monitoring

                

                    Automatically notify users when their requested content isn't available yet due to release dates or quality profile settings.
                

                
                

                    How it works:

                    
                        1️⃣ Request approved → Quality check after 10 seconds

                        2️⃣ If no quality found → "Waiting" email queued with delay

                        3️⃣ Radarr/Sonarr Grab webhook → Cancels waiting email

                        4️⃣ Download completes → Sends "Available" email

                        

                        📅 Coming Soon: Movies/shows not yet released (sends premiere date immediately)

                        ⏳ Quality Waiting: Content available but not in requested quality (delayed, can be cancelled)
                    
                

                
                

                    

                        
                        
                    

                    
                    

                        

                            
                            
                            Background check for all pending requests
                        

                        
                        

                            
                            
                            Delay before sending (allows cancellation)
                        

                    

                

                
                

                    📡 Required Webhooks:

                    
                        For this feature to work properly, configure webhooks in Sonarr/Radarr:

                        • On Grab - Cancels quality waiting when download starts

                        • On Import - Sends available notification when download completes
                    
                

            


            
            

                
🚨 Issue Auto-fix

                

                    When users report issues in Seerr (bad audio, wrong subtitles, corrupted video, etc.), the portal can automatically blacklist the current file and trigger a new search in Sonarr/Radarr.
                

                
                

                    How it works:

                    
                        1️⃣ User reports issue in Seerr → Webhook fires

                        2️⃣ Portal logs the issue and takes action based on mode

                        3️⃣ Current file is blacklisted + new search triggered

                        4️⃣ When new file downloads → User gets "Issue Resolved" email

                        

                        If the issue persists, the user can report again in Seerr.
                    
                

                
                

                    
                    
                    In Manual mode, use the Issues tab to review and trigger fixes.
                

                
                

                    📡 Required Seerr Webhook:

                    
                        Enable Issue Created webhook event in Seerr → Settings → Notifications → Webhook

🔄 Reconciliation & Issue Cleanup

                

                    The reconciliation worker periodically checks for missed webhooks, orphaned episodes, and stale issues that need cleanup.
                

                
                

                    How it works:

                    
                        1️⃣ Missed Episodes — Scans Sonarr/Radarr for downloaded content that never triggered a webhook

                        2️⃣ Orphaned Tracking — Finds tracked episodes with no matching notification and creates one

                        3️⃣ Stale Issues (Fixing) — Issues stuck in "fixing" where the import webhook never came back

                        4️⃣ Stale Issues (Reported) — Issues that were never auto-fixed and remain in "reported"

                        5️⃣ Abandoned Issues — Issues stuck for too long with no resolution → marked as failed
                    
                

                
                

                    

                        
                        
                        How often to scan for missed webhooks & stale issues
                    

                    

                        
                        
                        How long to wait before checking "fixing" issues for resolution
                    

                

                
                

                    

                        
                        
                        How long before checking "reported" issues that were never fixed
                    

                    

                        
                        
                        Mark issues as failed if no resolution after this long
